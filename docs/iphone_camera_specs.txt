## What must be tuned for each change to take effect (the non-obvious preconditions)

#1  Explicit format. sessionPreset MUST be `.inputPriority` (NOT a resolution preset like
    .hd1920x1080). Then pick a format from `device.formats` and assign `device.activeFormat`
    inside a lockForConfiguration block. SIDE EFFECT (free): using activeFormat instead of
    a preset makes AVCaptureDevice turn videoHDR OFF automatically — presets turn it ON.
    Selection filter we use: 1920x1080, a frame-rate range reaching >=60, prefer isVideoBinned
    (big binned pixels = more ISO headroom, less processing).

#2  60 fps. Set BOTH `activeVideoMinFrameDuration` and `activeVideoMaxFrameDuration` to
    CMTime(value:1, timescale:60), in a lock block. REQUIRES #1: the default preset's
    format tops out at 30 fps, so the format must be chosen first (a 60-capable one).

#3  videoHDR off (explicit). `automaticallyAdjustsVideoHDREnabled = false` FIRST (else the
    system may re-enable HDR), then `isVideoHDREnabled = false` — but ONLY if
    `activeFormat.isVideoHDRSupported`. Lock block. Largely redundant with #1 (which already
    disabled it), which is why it read as ~neutral.

#4  Global tone mapping on. `isGlobalToneMappingEnabled = true`, guarded by
    `activeFormat.isGlobalToneMappingSupported`. HARD PRECONDITION: GTM and videoHDR are
    MUTUALLY EXCLUSIVE — setting GTM true THROWS if videoHDR is enabled. So you must set
    `automaticallyAdjustsVideoHDREnabled = false` first (that one line of #3 is mandatory
    here). Lock block.

#5  Low-light boost off. `automaticallyEnablesLowLightBoostWhenAvailable = false`, guarded
    by `isLowLightBoostSupported`. Lock block. Only has any effect when the scene is dark
    enough for iOS to engage the boost — on a lit screen it likely never engages, so this
    is effectively a no-op (its "worse" number was probably thermal/noise).

#6  Geometric distortion correction off. `isGeometricDistortionCorrectionEnabled = false`,
    guarded by `isGeometricDistortionCorrectionSupported`. Lock block. Stops the per-frame
    barrel-undistort resample; straight screen edges then bow slightly near frame margins
    (refine_quad's line fits absorb it).

#7  Video stabilization off. THIS ONE IS NOT ON THE DEVICE. It is a property of the
    AVCaptureConnection: `output.connection(with: .video)?.preferredVideoStabilizationMode
    = .off`, guarded by `connection.isVideoStabilizationSupported`. Set it OUTSIDE the
    device lock block (connection config, not device config). Easy to miss because every
    other switch here is a device property.

#8  Short shutter. Replaces the plain `exposureMode = .locked` with
    `setExposureModeCustom(duration:iso:completionHandler:)`. duration = min(current, 1/125)
    ~= 8 ms; ISO must be scaled UP by (oldDuration/newDuration) to preserve brightness AND
    clamped to `activeFormat.minISO ... maxISO`. Lock block, in settleAndLock AFTER the
    autofocus has converged (so we read the auto-picked exposure as the baseline to shorten
    from). Pairs with #2 (only worth the noise cost when 60 fps capture + high display fps
    make blend the dominant error). The tripled ISO (~90->~280) raises sensor noise, which
    used to fragment the geometry ring-blob — now absorbed by the despeckle + bezel-reject
    fixes in Geometry.swift, so #8 no longer breaks the locate step.

Bottom line for the next reader: adopt #1+#2, skip #3-#7, keep #8 only for the high-fps
future — and go fix the decode-worker throughput (ANE path), because THAT is what caps
the transfer, not the camera.
